## 0. 核心判断

这个系统不是单一选股因子，而是一个“市场状态机 + 板块轮动 + 自适应选股 + 仓位控制 + 信号回测验证”的日度/盘中复盘系统。

总框架可以抽象为：

`宏观流动性 -> 市场周期定位 -> 板块/风格轮动 -> 个股选股路径 -> 仓位管理 -> T+1/T+2 信号验证`

PDF 中已有 dashboard 形态，但没有完整源码和精确公式。可以复现系统结构、数据管线和大部分指标；若要做到数值完全一致，需要补齐或反推以下内容：

- `SC30/SC3/RSV` 的精确定义，是简单 RSV、分位数强度，还是通达信自定义强度序列。
- `FFD/KJHU`、`CS1/CS2/CS3`、`买讯过滤` 的公式。
- 大板块、细分主题、风格标签的具体成分股口径。
- `市场风险评分` 中胜率、赔率、权重、相似度的精确加权方式。
- `v5 状态机` 的全部节点转移条件。

## 2. 每个框架的最小复现数据

### 2.1 市场与指数层

必需：
- 交易日历。
- 指数日线：上证、深证、创业板、平均股价或等权全 A 指数。
- 指数盘中：1 分钟或 5 分钟 OHLCV，用于盘中 dashboard。
- 指数 CCI 所需 OHLC，周期至少覆盖 84 日以上。

Wind/数据库字段建议：
- `trade_date`
- `symbol`
- `open/high/low/close/pre_close/pct_chg`
- `volume/amount`

### 2.2 个股层

必需：
- 全 A 股票日线 OHLCV。
- 盘中 1m/5m bars 或实时 quote。
- 成交额、成交量、换手率、流通市值/自由流通市值。
- 涨跌停价格、是否涨停/跌停、是否 ST、是否停牌。
- 个股所属行业、概念、主题、大板块、风格标签。

用途：
- 计算涨跌家数、涨停/跌停、MA 站上率。
- 计算板块/主题等权收益和强度。
- 计算容量抱团成交额 top 股票。
- 过滤停牌、ST、新股、流动性不足标的。

### 2.3 板块、主题、风格层

必需：
- 大板块：科技、新能源、消费、周期、金融大盘、权重、高股息等。
- 细分主题：光刻机、存储芯片、CHIPLET、CPO、消费电子、锂矿、煤炭等。
- 每日成分股映射，最好保留历史成分。
- 板块/主题 OHLCV，可直接取 Wind 板块指数，也可用成分股等权/流通市值加权合成。

核心派生字段：
- `sector_ret_1d`
- `sector_ret_3d/5d/20d`
- `sector_amount`
- `sector_turnover`
- `SC3`
- `SC30`
- `SC60`
- `RSV`
- `RSV_mom`
- `phase`
- `risk_level`

### 2.4 广度与情绪层

必需：
- 上涨/下跌家数。
- 涨停/跌停家数。
- 昨日涨停今日收益。
- MA5/MA13/MA50 站上率。
- aligned_up/aligned_down/short_diverge/trend_turning/mixed 等 alignment 分类。
- breadth 比例，例如上涨家数:下跌家数。

说明：
- `alignment` 不是 Wind 直接字段，需要按自定义规则由个股/板块趋势状态派生。
- PDF 中使用 `aligned_up 63->25`、`aligned_down 2->32`，更像是细分板块状态计数，而不是股票家数。

### 2.5 历史回测与状态库

必需：
- 每个交易日收盘后的全量特征快照。
- 每个交易日未来 1/5/10/20/40/60 日收益。
- 每个信号触发后的 T+1/T+2/T+3 结果。
- 每个状态的样本数、胜率、均值、赔率、最大回撤等。

建议表：
- `daily_market_snapshot`
- `daily_sector_snapshot`
- `signal_event`
- `signal_outcome`
- `state_forward_return_stats`
- `similar_state_neighbors`

## 3. 关键指标可复现方式

### 3.1 SC30 / SC3 / SC60

PDF 定义：
- `SC30`：30 日板块强度，中期趋势。`>80` 偏热/强势，`<25` 冰点/超卖。
- `SC3`：3 日短期强度，反映短期加速、减速、短期顶底。
- `SC60`：在板块详情中出现，用于更长周期趋势。

可复现候选：
- 方案 A：RSV 口径，`(当前值 - N日最低) / (N日最高 - N日最低) * 100`。
- 方案 B：N 日收益分位，按全市场主题横截面排序转 0-100。
- 方案 C：板块价格、成交额、广度加权后的综合强度。

建议：
- 第一版用方案 A 或 B 实现。
- 用 PDF 中 2026-05-19/2026-05-20 的数值校准，看哪个口径最接近。

### 3.2 流动性指标

PDF 阈值：
- `<45`：匮乏。
- `45-90`：充足，但可能是高位震荡派发。
- `>90`：溢出。

输入：
- 全市场总成交额。
- 平均换手率。
- 成交额相对过去 N 日分位。

盘中处理：
- 不能直接拿 10:30 的成交额和收盘阈值比较。
- 需要做时间进度归一化，例如用历史同一分钟成交占全天比例估算全天成交额。

### 3.3 MST 市场宽度

PDF 输出：
- MST-5：短线。
- MST-13：波段。
- MST-50：趋势。
- MA5/MA13 站上率。

可复现候选：
- `MST-N = 全市场个股收盘价站上 N 日均线的比例 * 100`
- 或者 `MST-N = 个股 N 日趋势为正的比例 * 100`

建议：
- 第一版直接用站上均线比例实现。
- 同时保留 MA5/MA13/MA50 站上率原始字段，便于校准。

### 3.4 容量抱团强度

PDF 输出：
- 绝对抱团。
- 强度，例如 `54.9/25`。
- 占比，例如 `11.13%`。
- 天数，例如第 25 天。
- 衰减，例如 `44.9`。

输入：
- 个股成交额。
- 板块/主题归属。
- top 成交额股票在全市场成交额中的占比。
- top 主线成交额占全市场或全样本成交额的占比。

可复现候选：
- `crowding_strength = 主线 top N 股票成交额占比的历史分位`
- `crowding_share = 主线 top N 股票成交额 / 全市场成交额`
- `decay = 主线强度峰值 - 当前强度` 或 `top 主线短期动量回落幅度`

### 3.5 RRG 轮动

四象限：
- 右上：领涨，RS-Ratio 强、RS-Momentum 强。
- 左上：转强，RS-Ratio 弱、RS-Momentum 强。
- 右下：转弱，RS-Ratio 强、RS-Momentum 弱。
- 左下：落后，双弱。

输入：
- 板块相对基准收益。
- 相对强度平滑序列。
- 相对强度动量。

第一版：
- `RS-Ratio = sector_price / benchmark_price` 的滚动 Z-score。
- `RS-Momentum = RS-Ratio - RS-Ratio_MA` 或其斜率。

### 3.6 周期论与价格-情绪模型

PDF 阶段：

`酝酿 -> 抱团 -> 高潮早段/中段/后段/顶峰 -> 主流轮动 -> 主流缩容 -> 缩容逼空 -> 瓦解`

关键触发变量：
- 主线 RSV 是否创新高。
- 中位接力梯队是否崩盘。
- 涨停是否跌破关键阈值，如 100、50、30。
- 跌停是否上升。
- breadth 是否翻空。
- alignment 是否翻空。
- 创业板是否破位。
- 总成交额是否低于缩容阈值或高于放量回归阈值。

PDF 中的模型语言：
- `a 模型放量回归`
- `b 模型疲态/末段 V反失败`
- `c 模型萌芽：价格倒追情绪`

这个框架可复现，但需要把文本规则结构化为状态机规则。

## 5. Wind 接入映射

Wind 可以作为数据源，但不要把业务逻辑写死在 Wind API 上。建议用适配层：

`Wind/TDX/本地数据库 -> raw tables -> feature engine -> state engine -> dashboard`

Wind 需要拉取的数据类别：
- A 股股票基础信息。
- 股票日线行情。
- 股票分钟行情或实时行情。
- 指数日线/分钟行情。
- 行业、概念、主题成分。
- 股票停复牌、ST、涨跌停价。
- 流通市值、换手率。

如 Wind 的主题板块口径和 PDF 中的通达信主题不一致，需要维护一套自定义 `sector_master` 和 `sector_constituent`，否则 SC30/热力图/选股池会偏差很大。

## 7. 复现实施路线

### 第一阶段：数据底座

目标：
- 建表。
- 接 Wind 或现有数据库。
- 固化大板块、主题、风格成分。
- 拉取至少 3-5 年历史数据。

输出：
- 原始行情表。
- 板块成分表。
- 可重复运行的数据更新任务。

### 第二阶段：指标引擎

目标：
- 先实现 CCI、流动性、MST、MA 站上率、涨跌停、breadth。
- 再实现 SC3/SC30/SC60、RSV、RRG、容量抱团。

输出：
- `feature_daily`。
- 盘中 `feature_intraday_snapshot`。

### 第三阶段：状态机与回测

目标：
- 实现市场状态机和周期阶段规则。
- 对每个状态计算未来收益。
- 生成风险评分和历史相似样本。

输出：
- `market_state_daily`。
- `state_forward_return_stats`。
- `similar_state_neighbors`。

### 第四阶段：自适应选股

目标：
- 先用可解释代理规则替代未知的 `FFD/KJHU/CS1/CS2/CS3`。
- 接入真实通达信公式或通过历史样本反推后替换。

输出：
- 每日/盘中股票池。
- 选股路径解释。
- 信号验证记录。

### 第五阶段：Dashboard

目标：
- 用 Streamlit/Dash/Shiny/React 任一技术栈实现盘中监控。
- 定时刷新快照。
- 日终保存 PDF/HTML 复盘。

输出：
- 盘中 dashboard。
- 日终复盘报告。

## 8. 最大风险点

1. 板块口径风险：Wind 概念、申万行业、通达信概念的成分差异会导致强度数值明显不同。
2. 公式缺失风险：PDF 没有给出 `FFD/KJHU/CS1/CS2/CS3` 的完整定义。
3. 盘中归一化风险：成交额、换手率、强度在盘中必须做时间进度校正。
4. 过拟合风险：风险评分和相似时期必须保留样本数、胜率、均值、赔率，不能只看单一胜率。
5. 状态机漂移风险：周期论部分带有人工规则语言，需要逐步结构化，不宜一次性硬编码过多主观判断。

## 9. 第一版最小可用产品建议

MVP 不要一开始追求完全复刻 PDF。优先顺序：

1. CCI、流动性、MST、MA 站上率。
2. 大板块/主题 SC30、SC3、热力图。
3. 容量抱团成交额集中度。
4. 风格轮动和 RRG。
5. 市场状态机第一版。
6. 历史回测评分。
7. 自适应选股路径。
8. 周期论文本自动生成。

这样可以先得到可用的盘中 dashboard，再逐步校准到 PDF 风格。

### 10.3 新增框架 2：板块近期变化 Top7 排名系统

文章 2 和文章 8 都使用同一套排名框架：127 个板块，计算 `几日涨幅` 参数对应的 N 日涨幅，然后逐层取前 7 名。

核心框架：
- `ZS_i = 板块收盘价 / N日前板块收盘价 - 1`，再转为百分比。
- `LCW1` 是全部 127 个 `ZS_i` 的最大值，即第 1 名。
- `LCW2` 是剔除第 1 名后的最大值。
- 同理迭代到 `LCW7`，得到 Top7。
- 用 `ZQBK1` 到 `ZQBK7` 存储排名板块名称。
- 显示今日排名、昨日排名、前日排名等，用于观察排名迁移。
- 取今日 Top3、昨日 Top3、前日 Top3，用 `HORCALC` 对板块内部做横向统计，再用 20 日窗口求和和平滑 EMA 展示走势。

### 10.4 新增框架 3：AI 板块强度三层监测体系

文章 8 对 AI 板块强度的文字解释比较完整，可以抽象为三层：

1. 广度扫描：最多支持 127 个细分板块，覆盖 AI 核心和泛科技延伸方向。
2. 深度聚焦：通过 Top7 排名筛选当日最强阵营，再剔除已入选板块寻找替补。
3. 时空对比：同时显示今日、昨日、前日排名，并叠加平滑曲线识别持续性。

这个框架与 PDF 中的“科技主线、光刻机/存储芯片/CHIPLET/CPO 等顶级 RSV 创新高”高度一致。区别是：
- PDF 用 `SC30/SC3/RSV/mom/phase` 描述周期位置。
- 公众号源码用 `N日涨幅排名 + Top7迁移 + 板块内部横向统计` 描述主线强弱。

建议在复现系统中同时保留二者：
- `sector_strength_score`：SC/RSV 体系。
- `sector_rank_score`：N 日涨幅排名体系。
- `sector_persistence_score`：连续进入 Top7 的天数。
- `sector_rotation_event`：新晋、掉出、连续、回归。

### 10.5 新增框架 4：趋势牛卡位买点

文章 3 是个股趋势买点模块，适合放进“自适应选股系统”的趋势路径。

核心逻辑：
- `大牛卡位`：用近 10 日最高收盘价的 30 日 EMA，再乘 0.98，形成趋势卡位线。
- `长波回档`：用 17 日 EMA 乘 0.96，形成回档线。
- 前置条件包含：近期涨停、放量、均线快速上行、跳空/突破、回踩短均线等。
- 普选：信号出现且股价在长波回档线上方。
- 精选：信号出现且长波回档线在大牛卡位线上方，代表趋势更强。

复现数据：
- 个股 OHLCV。
- 涨停识别。
- MA5/MA10/MA20/MA30/MA60。
- VOL 的 120 日均量。
- 跳空缺口。
- 最近 5/10/15/20 日是否出现涨停或放量事件。

接入建议：
- 作为 `buy_signal_flags.trend_position_buy`。
- 对容量抱团股票池、主线板块内股票优先打分。
- 不应单独全市场使用，最好受市场状态机和板块主线过滤。

### 10.6 新增框架 5：高胜率抄底指标

文章 5 和文章 6 是同一套超跌反弹系统，一个是图形指标，一个是选股公式。

信号分层：
- 绝底：34 日乖离极度负值，同时当日振幅、实体涨幅、日涨幅都足够强，文章标注概率 95%。
- 超高概率：价格相对 21 日 EMA 的偏离率上穿 -20，文章标注概率 87%。
- 高概率：价格相对 24 日 MA 的偏离率上穿 -20，文章标注概率 81%。
- 中高概率：近 3 日低点创 120 日新低，同时当日涨幅不低，文章标注概率 68%。
- 中概率：近 3 日低点创 60 日新低，同时当日涨幅不低，文章标注概率 60%。

文章 6 还包含：
- 500/250/90 日 HHV 和 LLV 的多周期动态支撑/压力带。
- 基于低点波动的能量计算。
- “能量坑/能量集”可视化。

复现数据：
- 个股 OHLC。
- MA/EMA。
- 60/90/120/250/500 日最高最低。
- 当日振幅、实体涨幅、涨跌幅。

注意：
- 文章中的概率是作者标注，不等于我们数据库回测后的胜率。
- 接入时必须用本地历史数据重算每个信号的 1/5/10/20 日胜率、均值、回撤。
- 这个模块只适合“超跌反弹/冰点修复”状态，不适合在主线高潮期无过滤地使用。

### 10.7 新增框架 6：擒龙捉妖情绪龙头系统

文章 7 是个股情绪龙头模块，覆盖涨停板、攻击资金、妖股、盈利加仓等信号。

核心子模块：
- 涨停识别：以前收盘价为基准判断接近 10% 涨停且收最高。
- 夺命板：昨日缩量、今日放量但未超过昨日 2 倍，同时股价上穿 21 日均线并涨停。
- 工作/度假线：用 `(L+H+2C)/4` 的 14 日和 25 日 EMA 作为趋势状态。
- 动量摆动：用 30 日 HHV/LLV 归一化后做 EMA，形成类似 KDJ/RSV 的强弱指标。
- BBI BOLL：3/6/12/24 均线均值叠加标准差通道。
- 攻击资金：用短 EMA 与长 EMA 差值构造资金攻击强度。
- 修炼成妖/妖王出世：攻击资金暴增叠加连续强涨和涨停。
- 深水掘金：25 日区间振荡结构反转。
- 成本线：短线成本、中线成本、长线成本。
- 盈利加仓：强涨、收盘接近高点、价格突破兑现线且内在动力足够强。

复现数据：
- 个股 OHLCV、成交额。
- 涨停/跌停价。
- MA/EMA/STD。
- 成交均价 `AMOUNT / VOL`。
- 成本分布函数。通达信 `COST(84)` 在 Wind 中没有直接等价，需要用筹码分布模型或成交量价分布近似。

接入建议：
- 作为 `buy_signal_flags.emotion_leader`、`profit_add_signal`、`leader_acceleration_signal`。
- 需要市场情绪过滤：涨停数量、连板高度、主线板块排名、成交额集中度。
- 不建议在下行初期直接追涨使用，除非 dashboard 显示主线逼空或情绪修复。

### 10.8 新增框架 7：缠论主图自动结构系统

文章 4 是图形结构模块，重点不是横截面选股，而是自动标注趋势结构。

核心子模块：
- K 线包含关系处理。
- 分型识别。
- 笔和线段识别。
- 中枢上下沿 `ZG/ZD`。
- 一买、类二买、二买、三买。
- 一阳定势、收敛、强势。
- 涨停线和涨停幅。

复现难点：
- 文章源码大量使用通达信 `TDXDLL3`，这是黑箱 DLL。
- 如果脱离通达信，需要重新实现缠论算法，包括包含关系、分型、笔、线段、中枢。

接入建议：
- 第一版 dashboard 不把缠论作为核心状态机输入。
- 可以作为个股详情页的结构确认模块。
- 数据库中可落表：`fractal_points`、`chan_bi`、`chan_segment`、`chan_zhongshu`、`chan_buy_points`。

## 11. 公众号内容对复盘系统的修正

公众号文章让几个原先不明确的地方更清楚：

1. 板块热力图不是纯 SC30/SC3，它至少有一个明确版本是“板块 RSI 的 EMA(5)”。
2. 板块近期变化不是复杂机器学习模型，而是“127 板块 N 日涨幅 Top7 + 历史排名迁移 + Top3 横向统计走势”。
3. AI 板块强度监测是主线监控器，和 PDF 中的科技主线/顶级 RSV 监控可以合并。
4. 自适应选股系统可以拆成三类底层信号：
   - 趋势卡位：适合主线强趋势。
   - 抄底反弹：适合冰点/超跌修复。
   - 情绪龙头：适合涨停生态和主线逼空。
5. 通达信依赖项必须单独替代：
   - `CALCSTOCKINDEX`：用板块指数或成分股合成指标替代。
   - `HORCALC`：用板块成分横截面统计替代。
   - `TDXDLL3`：用自研缠论结构算法替代。
   - `COST`：用筹码/成交量价分布模型替代。
   - `GETNAMEOFCODE`：用本地板块字典替代。

## 13. 更新后的 MVP 顺序

更合理的落地顺序应调整为：

1. 数据底座：股票、指数、板块、概念、涨跌停、成交额、换手率。
2. 板块热力图：先复刻 50 板块 RSI-EMA 热力值。
3. 板块近期变化：复刻 127 板块 N 日涨幅 Top7。
4. AI/科技主线监测：复刻 127 AI/泛科技板块强度。
5. PDF 核心 dashboard：CCI、流动性、MST、仓位。
6. 容量抱团：成交额集中度、主线 top 股票。
7. 个股信号：趋势卡位、抄底、情绪龙头。
8. 回测验证：把所有信号写入事件表，重算胜率。
9. 状态机：用历史结果校准 v5 节点和周期论文本。
10. 缠论结构：最后接入，作为个股详情增强模块。
